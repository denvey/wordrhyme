## ADDED Requirements

### Requirement: Rich Text Editor Package

The system SHALL provide a `@wordrhyme/editor` package wrapping EditorJS with CMS-specific configuration. The editor SHALL output structured JSON (EditorJS OutputData format), not HTML. The editor SHALL integrate with the existing `AssetService` for image uploads.

Standard blocks included:
- Paragraph, Heading (H1-H6), List (ordered/unordered), Quote, Code, Delimiter
- Table, Embed (YouTube, Vimeo, etc.)
- Image (integrated with AssetService upload)
- Gallery (multi-image, reuses asset picker)

```typescript
interface RichTextEditorProps {
  initialData?: OutputData;
  onChange?: (data: OutputData) => void;
  onReady?: (editor: EditorJS) => void;
  readOnly?: boolean;
  placeholder?: string;
  tools?: Record<string, ToolConstructable>; // additional custom blocks
}
```

#### Scenario: Editor initializes with standard blocks
- **WHEN** `<RichTextEditor />` is rendered without custom tools
- **THEN** all standard blocks (paragraph, heading, list, quote, code, image, table, embed, delimiter) are available
- **AND** the editor outputs structured JSON on change

#### Scenario: Image block uploads via AssetService
- **WHEN** a user adds an image block and selects a file
- **THEN** the file is uploaded via the existing asset tRPC endpoint
- **AND** the image URL in the block data points to the CMS asset path
- **AND** thumbnail variants are auto-generated by ImageProcessorService

#### Scenario: Editor round-trips data correctly
- **WHEN** `initialData` is provided with valid EditorJS OutputData
- **THEN** the editor renders all blocks correctly
- **AND** calling `onChange` returns equivalent OutputData (no data loss)

### Requirement: Plugin-Contributed Editor Blocks

Plugins SHALL be able to contribute custom EditorJS block tools via the `editor.blocks` array in `manifest.json`. Custom blocks SHALL be loaded at editor initialization via Module Federation.

```json
{
  "editor": {
    "blocks": [
      {
        "name": "product-card",
        "exportName": "ProductCardBlock",
        "label": "Product Card",
        "icon": "ShoppingBag"
      }
    ]
  }
}
```

#### Scenario: Plugin block appears in editor toolbar
- **WHEN** a plugin declares an editor block in its manifest
- **AND** the plugin is enabled
- **THEN** the custom block type appears in the editor's block insertion menu
- **AND** the block component is loaded via Module Federation remote entry

#### Scenario: Plugin block renders in read-only mode
- **WHEN** content containing a plugin-contributed block is viewed in `<RichTextRenderer>`
- **AND** the contributing plugin is enabled
- **THEN** the block renders using the plugin's component
- **AND** if the plugin is disabled, a fallback placeholder is shown
