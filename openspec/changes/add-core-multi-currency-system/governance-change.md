## 修订内容

### 3.1 核心原则（修改）

**原文**：
```markdown
> **金额永远与币种绑定，系统不做隐式换算。**
```

**修改为**：
```markdown
> **金额永远与币种绑定。系统提供显式换算服务，但结算金额必须与实时汇率绑定记录。**
```

### 3.4 严禁行为（修改）

**原文**：
```markdown
❌ Core 自动换汇
❌ 前端计算金额
❌ 插件处理币种
❌ Billing 写死某种货币
```

**修改为**：
```markdown
### 3.4 换算规则（v0.2 更新）

#### 允许的行为
✅ Core 提供汇率换算服务
✅ 前端使用缓存汇率展示参考价格（必须标注"参考价格"）
✅ 管理员手动设置或自动同步汇率

#### 严禁的行为
❌ 插件处理币种或汇率
❌ Billing 写死某种货币
❌ 结算时使用缓存汇率（必须实时获取）
❌ 交易记录缺少汇率快照
❌ 前端展示不标注"参考价格"
```

### 3.5 新增：汇率管理规范

```markdown
### 3.5 汇率管理规范（v0.2 新增）

#### 3.5.1 汇率数据模型

汇率表必须包含：
- base_currency: 基准货币
- target_currency: 目标货币
- rate: 汇率（精度 8 位小数）
- source: 来源（'manual' | 'api:provider_name'）
- effective_at: 生效时间
- expires_at: 过期时间（可选）
- organization_id: 组织隔离

#### 3.5.2 汇率来源

| 模式 | 描述 | source 值 |
|------|------|-----------|
| 手动 | 管理员设置 | 'manual' |
| 自动 | API 同步 | 'api:exchangerate-api' 等 |

#### 3.5.3 版本控制

- 汇率表必须有全局版本号
- 前端缓存必须基于版本号失效
- 版本号变更时广播通知

#### 3.5.4 审计要求

每次汇率变更必须记录：
- 操作人（管理员 / 系统）
- 变更时间
- 变更前后值
- 变更来源

### 3.6 结算规范（v0.2 新增）

#### 3.6.1 结算必须使用实时汇率

结算时（创建支付意向）必须：
1. 获取实时汇率（非缓存）
2. 计算结算金额
3. 在交易记录中保存汇率快照

#### 3.6.2 交易记录必须包含

| 字段 | 描述 |
|------|------|
| base_currency | 基准货币 |
| base_amount | 基准金额 |
| settlement_currency | 结算货币 |
| settlement_amount | 结算金额 |
| exchange_rate | 结算时汇率 |
| exchange_rate_at | 汇率获取时间 |

#### 3.6.3 展示与结算差异

允许展示价格与结算价格存在差异，但：
- 差异由汇率波动引起
- 结算确认页必须显示最终金额
- 用户必须确认后才能支付
```

## 变更理由

1. **业务需求**：多货币展示需要汇率换算
2. **用户体验**：用户希望看到本地货币价格
3. **管理便利**：管理员只需维护一套基准价格
4. **控制边界**：
   - 汇率仍由 Core 管理（非插件）
   - 结算仍使用实时数据（非缓存）
   - 审计仍完整可追溯
