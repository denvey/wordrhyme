# Nebula CMS — Plugin Marketplace Governance (v0.1)

> 本文档定义 **Nebula CMS 插件市场的治理规则（Governance Contract）**。
>
> 该文档用于约束：
>
> * 官方插件市场（Marketplace）
> * 插件作者（Developers）
> * 平台运营方（Platform Operator）
>
> 适用于 **开源 Nebula（OSS）** 与 **Nebula SaaS 平台** 两种形态。

---

## 1. 治理目标（Governance Goals）

插件市场的存在不是为了“分发代码”，而是为了：

1. 建立 **可信插件生态**
2. 保障 **最终用户的业务连续性**
3. 保护 **插件作者的商业权益**
4. 明确 **平台的最终治理权与责任边界**

---

## 2. 插件市场基本原则（Marketplace Invariants）

以下原则不可被违反：

1. **插件市场是“发行渠道”，不是运行时依赖**
2. **已安装插件的运行权优先于市场状态**
3. **平台对市场拥有最终裁决权**
4. **治理规则优先于商业关系**

---

## 3. 插件状态模型（Plugin Lifecycle States）

插件在市场中的状态如下：

```ts
type MarketplacePluginState =
  | 'draft'
  | 'published'
  | 'archived'
  | 'suspended'
  | 'removed';
```

### 3.1 状态语义说明

| 状态        | 说明              |
| --------- | --------------- |
| draft     | 未发布，仅作者可见       |
| published | 正常分发与安装         |
| archived  | 停止新安装，但已安装实例可运行 |
| suspended | 因违规或风险被临时冻结     |
| removed   | 被永久移除，仅保留审计记录   |

---

## 4. 插件上下架与兼容性治理

### 4.1 上架要求（原则级）

市场插件必须：

* 声明支持的 Nebula Core 版本范围
* 明确插件数据策略（引用 Plugin Data Governance）
* 明确权限声明（引用 Permission Contract）

---

### 4.2 Core 不兼容时的处理

当插件与新 Core 版本不兼容时：

```md
- 插件可被标记为 archived
- 已安装实例不被强制卸载
- 插件作者可选择更新或放弃维护
```

平台 **不得** 因升级 Core 而强制破坏用户现有业务。

---

## 5. 插件收费与 License 失效治理

### 5.1 收费插件的运行原则

* License 仅影响 **安装 / 更新 / 支持服务**
* License 失效 ≠ 插件立即停止运行

---

### 5.2 License 失效后的行为（推荐）

```md
- 插件继续运行（grace mode）
- 禁止新安装
- 禁止版本升级
- 可限制高级功能（由插件自行实现）
```

> 平台不得在运行时强制破坏用户已有功能。

---

## 6. 插件作者弃坑与平台兜底

### 6.1 插件作者责任

插件作者应：

* 提供合理维护周期
* 声明支持的 Core 版本
* 遵守市场规则与合规要求

---

### 6.2 作者失联 / 弃坑的处理

```md
- 插件可被标记为 archived
- 平台可接管发行（不接管代码所有权）
- 已安装用户拥有继续使用权
```

平台 **不承诺** 接管插件维护责任，但承诺 **不主动破坏用户系统**。

---

## 7. 安全、违规与紧急处理

### 7.1 可触发紧急治理的情况

* 恶意行为
* 数据滥用
* 严重安全漏洞
* 法律或合规风险

---

### 7.2 平台治理权（强制）

平台保留以下权利：

```md
- suspend plugin distribution
- revoke licenses
- force-disable plugins (SaaS only)
```

> OSS 环境下，平台仅提供建议，不具备强制执行权。

---

## 8. OSS 与 SaaS 的治理边界

| 能力   | OSS  | SaaS     |
| ---- | ---- | -------- |
| 插件安装 | 完全自由 | 受平台规则约束  |
| 插件下架 | 无效   | 平台可控     |
| 强制禁用 | 不支持  | 支持（极端情况） |

---

## 9. 分成与商业条款（原则级）

* 插件市场可对交易进行分成
* 分成比例与结算方式由平台定义
* 商业条款 **不影响插件运行权**

---

## 10. 明确非目标（Non-Goals）

Nebula 插件市场 **不承诺**：

* 插件源码托管义务
* 插件商业成功保证
* 插件作者收益最低保障

---

## 11. 演进策略（Forward Compatibility）

* 市场治理规则可演进
* 已安装插件的运行权不可追溯性破坏
* 治理 Contract 高于商业协议

---

> **插件市场的本质不是“卖插件”，而是“卖稳定性与信任”。**
>
> 这是 Nebula 能成为长期平台的最后一道防线。
