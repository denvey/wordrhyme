# 权限内核规范 (Permission Kernel Specification)

## 新增要求

### 要求：白名单授权模型 (White-list Authorization Model)

权限内核应实现一种默认拒绝的授权模型。除非明确授予，否则所有能力都被拒绝。权限必须集中评估 (插件不能自行授权)。

#### 场景：拒绝未声明的能力
- **当** 用户尝试使用其角色中没有的能力时
- **那么** 权限检查返回 `false`
- **并且** 请求被拒绝并返回 403 错误
- **并且** 创建一条审计日志条目

#### 场景：授予声明的能力
- **当** 用户具有“admin”角色时
- **并且** “admin”角色具有 `core:users:manage` 能力
- **当** 用户调用 `permissions.can(user, 'core:users:manage', scope)` 时
- **那么** 权限检查返回 `true`

---

### 要求：租户限定的权限 (Tenant-scoped Permissions)

所有权限必须限定在租户作用域内。默认拒绝跨租户访问。权限检查必须在作用域中包含 `tenantId`。

#### 场景：授予同租户访问权限
- **当** 用户 A (租户 T1) 具有 `content:read` 能力时
- **并且** 用户 A 请求租户 T1 中的内容时
- **那么** 权限检查通过

#### 场景：拒绝跨租户访问
- **当** 用户 A (租户 T1) 具有 `content:read` 能力时
- **并且** 用户 A 请求租户 T2 中的内容时
- **那么** 权限检查失败
- **并且** 返回 403 错误

---

### 要求：能力格式 (Capability Format)

能力应遵循 `resource:action:scope` 格式。权限内核必须在权限检查期间验证能力格式。

#### 场景：有效的能力格式
- **当** 能力被定义为 `content:create:workspace` 时
- **那么** 格式验证通过
- **并且** 能力可以分配给角色

#### 场景：拒绝无效的能力格式
- **当** 能力被定义为 `invalid-format` (无效格式) 时
- **那么** 格式验证失败
- **并且** 返回一个错误

---

### 要求：权限缓存 (Permission Caching)

权限检查可以在单个请求内进行缓存以提高性能。缓存的权限不得在请求之间持久存在。当用户角色更改时，必须使缓存失效。

#### 场景：单个请求内的缓存
- **当** 在请求 1 中为用户 A 执行权限检查时
- **那么** 结果在请求 1 的持续时间内被缓存
- **当** 在请求 1 中再次检查相同的权限时
- **那么** 返回缓存的结果 (无需查询数据库)
- **当** 为同一用户启动请求 2 时
- **那么** 执行新的权限检查 (缓存已清除)

#### 场景：角色更改使缓存失效
- **当** 用户 A 的角色在请求 1 期间发生更改时
- **那么** 用户 A 的权限缓存被清除
- **并且** 后续检查将查询最新数据
